# probevelo

Work-around for RNA velocity inference from 10X Fixed RNA Profiling using probe sets.

## Installation

Only available in experimental stage on GitHub for now:

```bash
pip install git+https://github.com/mvfki/probevelo.git
```

## Usage

### CLI command

After installing the package, you can use the command line interface to count spliced and unspliced reads from a BAM file generated by 10X Fixed RNA Profiling, and verify the results against reference counts.

#### Count spliced and unspliced reads

**Standard mode** for typical datasets:

```bash
probevelo count \
    /path/to/10x/out/sample_alignment.bam \
    /path/to/10x/out/probe_set.csv \
    --output /path/to/output/splicing.h5ad \
    --threads 12
```

**Memory-efficient mode** for large datasets (6M+ barcodes):

```bash
probevelo count \
    /path/to/10x/out/sample_alignment.bam \
    /path/to/10x/out/probe_set.csv \
    --output /path/to/output/splicing.h5ad \
    --memeff \
    --max-mem 12.0 \
    --chunk-size 10000000 \
    --threads 12
```

#### Verify results

Verify that probevelo counts match 10X Genomics (spaceranger/cellranger) raw counts:

```bash
probevelo verify \
    /path/to/probevelo/output.h5ad \
    /path/to/10x/out/raw_feature_bc_matrix.h5
```

#### Command options

**Main command:**

```
$ probevelo
usage: probevelo [-h] [-v] {count,verify} ...

ProbeVelo: Work around for counting spliced and unspliced reads from FRP BAM files.

positional arguments:
  {count,verify}  Available commands
    count         Count spliced and unspliced reads from BAM file
    verify        Verify probevelo output against 10X raw counts

options:
  -h, --help      show this help message and exit
  -v, --version   Show program's version number and exit
```

**Count subcommand:**

```
$ probevelo count
usage: probevelo count [-h] [-t THREADS] [-o OUTPUT] [--memeff] [--max-mem MAX_MEM]
                       [--chunk-size CHUNK_SIZE] [--temp-dir TEMP_DIR] [-q]
                       BAM PROBE_SET

Count spliced and unspliced reads from probe-based FRP BAM files.

positional arguments:
  BAM                   Path to the BAM file containing RNA sequencing data. Must be indexed.
  PROBE_SET             Path to the probe set CSV file.

options:
  -h, --help            show this help message and exit
  -t THREADS, --threads THREADS
                        Number of threads to use for counting (default: 1).
  -o OUTPUT, --output OUTPUT
                        Output file path for the results (default: output.h5ad).
  --memeff              Use memory-efficient counting mode for large datasets (6M+ barcodes).
  --max-mem MAX_MEM     Maximum memory to use in GB for memeff mode (default: 8.0).
  --chunk-size CHUNK_SIZE
                        Number of entries to accumulate before flushing to disk in memeff
                        mode (default: 10,000,000).
  --temp-dir TEMP_DIR   Directory for temporary files in memeff mode (default: system temp).
  -q, --quiet           Suppress logging output (default: False).
```

**Verify subcommand:**

```
$ probevelo verify
usage: probevelo verify [-h] [-q] probevelo_h5ad tenx_h5

Verify that probevelo spliced+unspliced counts match 10X Genomics (spaceranger/cellranger) raw counts.

positional arguments:
  probevelo_h5ad  Path to probevelo output h5ad file
  tenx_h5         Path to 10X raw_feature_bc_matrix.h5 file (spaceranger or cellranger
                  output)

options:
  -h, --help      show this help message and exit
  -q, --quiet     Suppress info messages

Examples:
  # Verify output matches reference counts
  probevelo verify output.h5ad raw_feature_bc_matrix.h5
```

Note that **the BAM file must be indexed**, e.g. you must have `sample_alignment.bam.bai` in the same directory as the input file `sample_alignment.bam`.

### Python API

If you prefer to use the package programmatically, you can do so by importing the `probevelo` module in your Python script. The package provides a simple interface to load the probe set information and BAM information and perform the counting, returning an `anndata.AnnData` object with the results.

#### Standard counting

```python
import probevelo

probe_set = probevelo.probe_set('/path/to/10x/out/probe_set.csv')
bam_parser = probevelo.bam_parser('/path/to/10x/out/sample_alignment.bam', n_thread=32)

adata = bam_parser.count_splice(probe_set, adata=True)
```

#### Memory-efficient counting

For large datasets (6M+ barcodes), use the memory-efficient mode:

```python
import probevelo

probe_set = probevelo.probe_set('/path/to/10x/out/probe_set.csv')
bam_parser = probevelo.bam_parser_memeff(
    '/path/to/10x/out/sample_alignment.bam',
    max_mem_gb=12.0,
    chunk_size=10_000_000,
    temp_dir='/path/to/temp',
    n_thread=32
)

adata = bam_parser.count_splice(probe_set, adata=True)
```

#### Verification

Verify that probevelo counts match 10X reference counts:

```python
from probevelo import verify

verify(
    'probevelo_output.h5ad',
    'raw_feature_bc_matrix.h5'
)

# Alternatively, load the data into anndata objects first
# verify(
#    adata_spliced,

```

#### Output format

The returned `anndata.AnnData` object will contain:

- `adata.X`: the total raw counts, i.e. sum of spliced and unspliced counts
- `adata.layers['spliced']`: the spliced counts
- `adata.layers['unspliced']`: the unspliced counts
- `adata.obs`: Only the barcodes as the index
- `adata.var`: All gene IDs available in the probe set will be used as the index,
and the `gene_name` column will be mapped as an alternative column. Note that multiple
gene IDs can map to the same gene name, so we by default use the ID as the index.
